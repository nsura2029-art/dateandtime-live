Live endpoints we can hit today

  • GET /api/v1/time/now?tz=America/New_York — current time + offset + abbrev
  • GET /api/v1/time/sun?lat&lon&date — sunrise/sunset per city
  • GET /api/v1/dst/upcoming?tz= — next DST transition
  • GET /api/v1/holidays/today?country= — today's holiday per country
  • GET /api/v1/countries/US/working-hours — 9-5 work hours + days
  • GET /api/v2/search?q= — city search with FTS5 + aliases

🧭 Proposed menu structure

One /world-time/ hub with 4 sub-tools. Each gets its own URL so they're shareable + indexable for SEO.

dateandtime.live/world-time/
  • /world-time/personal/ — Personal World Clock
  • N cities, add/remove, save to localStorage, shareable URL
  • /world-time/meeting/ — Meeting Planner
  • 24-hour grid, working hours overlap, iCal export
  • /world-time/convert/ — Time Zone Converter
  • "3pm NY = ? in Tokyo", date-aware
  • /world-time/map/ — Time Zone Map
  • Interactive world map with day/night terminator

Or — single page with all 4 as tabs (faster, less SEO surface):

dateandtime.live/world-time/ (one URL, tabbed UI)
  • Tab 1: Personal
  • Tab 2: Meeting
  • Tab 3: Convert
  • Tab 4: Map

My pick Separate URLs — better SEO (4× the indexable pages), better shareability (someone can DM a meeting URL), better deep linking from blogs.
