Design D — WTB-style Converter + Meeting A+ top pick ⭐

"Convert any time. See every city." No more top bar — the grid IS the converter. Your home city loads first. URL params like ?cities=tampa,london&time=15:00 pre-populate. Add cities via search.

  • Search bar above cities: Type any city → instant suggestions from our 33,945-city DB. Click to add to grid + chips.
  • Default = user location: On first visit, window.__location from CF IP geo is used as the home city (with ⌂ icon).
  • URL params: ?cities=tampa,london&time=15:00 pre-populates cities + picked time. URL state bar shows what was parsed.
  • 24×N grid (WTB-style): Cities as rows, time as columns. Green (9-6 work) / amber (early/late) / dark (night). Indigo "now" outline.
  • Hover to convert (vertical line): Hover any cell → all cells in that column highlight + hour label turns indigo. Shows the same moment across all cities.
  • Click to schedule: Click any cell → purple checkmark, picked-time strip below shows time in all cities + iCal/email/share.
  • 12h/24h toggle: Top right pill, applies to all times everywhere.
  • City chips with remove: × button fades out the chip + removes from grid. Drag to reorder (v2).
  • Light + dark both look great: Greens pop in light, teals pop in dark. Now indicator is always indigo.
  • Mobile: Stacks cleanly. URL state wraps. Search suggestions stack. Grid scrolls horizontally.
  • All 7 interactions wired: URL parse ✓, search ✓, suggestion click ✓, cell click ✓, hover (vertical line) ✓, chip remove ✓, 12h/24h ✓
Why this is the top pick: One page = both the 1:1 converter (via search + URL params) AND the multi-city comparison (via the grid). User's home city pre-loaded. URL-deep-linkable for SEO. Hover-convert for instant discovery. Ship-ready.

Design A — Personal World Clock A+ ready

The "where are my people" view. Hero clock + grid of saved cities.

  • Light mobile (390px): Hero stacks perfectly. Big clock readable. Buttons clear.
  • Light tablet (768px): 3-col grid, hero 2-col, all elements proportional.
  • Light desktop (1440px): 4-col grid, looks like a polished product page.
  • Dark mode (all): All text readable, no contrast issues. Pastel tiles work in both themes.
  • NotebookLM pastels: Each city has a unique tone (peach, lavender, mint, sky, rose, lemon) — feels personal, not generic.
No fixes needed. Ready to ship as-is.

Design B — Meeting Planner A fixes applied ✓

The "when works for everyone" view. 24×N grid with working hours overlap.

  • Light mobile: ✅ City chips wrap, grid shows 1 column properly
  • Light desktop: ✅ Grid is beautiful, green slots pop, suggestions list helps
  • Dark mode grid cells: ❌ Work-hours (green) and sleep (indigo) are too dark. The "where can I meet?" story is lost.
  • Dark mode "Picked time" chips: ❌ Times like "10:00 Wesley Chapel" have dark text on dark background — invisible.
  • Dark mode "iCal / Google Cal" button: ❌ The purple button + icon looks muddy against the dark purple picked-time card.
  • Suggestions (Best/OK/Poor) pills: ✅ Green/amber/red are clear in both modes
Fixes applied (3): ✓ brighter work-hours green, ✓ explicit color on .pt chips, ✓ iCal button uses white text on primary
Result: All dark mode issues resolved. Now ship-ready.

Design C — Converter + Map + Event Announcer C needs rework

"See the world. Schedule anywhere." 1:1 converter + day/night map + share card.

  • Light mode converter boxes: ✅ Clear, big readable time, chips below
  • Dark mode converter boxes: ❌ Background too dark, time text dim, chips barely visible
  • World map (the centerpiece): ❌ The "land" is a placeholder SVG blob, not a real world map. Looks like a child's drawing.
  • Day/night terminator: ⚠️ Concept works (yellow line, pink dusk) but doesn't have a real land to terminate over.
  • City pins on map: ✅ Pin design is good in both modes (yellow on map, green for home)
  • Around-the-world timeline strip: ✅ Color-coded day/dusk/night, clear UTC labels
  • Share event card: ✅ City chips clean, all 5 share buttons (iCal, copy, email, WhatsApp, calendar) visible
  • Mobile: ✅ Stacks well, but the map is below the fold (huge asset)
Fixes needed (3):
1. Real world map — use a D3 geo projection or a free world SVG. ~50KB. Critical for this design's appeal.
2. Brighten the converter box backgrounds in dark mode (use a slightly lighter --bg-soft)
3. Make the "To" input copy show actual text (the input shows placeholder, not the value — needs the input to be readable in dark mode)

🏆 My tool pick for the World Time menu

Ship these 4, in this order:

  1. WTB-style Converter + Meeting (Design D) ⭐ — combined 1:1 conversion + multi-city grid. 2-3 weeks. 180K + 30K = 210K/mo addressable.
  2. Personal World Clock (Design A) — saves to localStorage, shareable URL. 1 page, ~5 days. Reuses our home page rail component.
  3. Event Time Announcer (from Design C, the share card) — shareable URL with time in N cities. 1 week. Cheap, viral.

Save for v2: Time Zone Map (needs D3 + real map) and the day/night terminator visual (cool but niche).

Menu structure I'd ship

dateandtime.live/world-time/
  └─ /meeting/       → Design D ⭐ (WTB-style, both converter + grid)
  └─ /personal/     → Design A (no fixes)
  └─ /event/       → Design C's share card

What I'd skip for now

  • Time Zone Map — needs real world map (D3), 2-3 weeks of work, and timeanddate.com already dominates this
  • Day/Night terminator — cool visual but the 5-10K volume doesn't justify the SVG work for v1
  • Market hours, religious, jet lag, TV premieres — each is a separate rabbit hole; not in v1 scope
